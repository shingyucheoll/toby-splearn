## 도메인 모델 만들기

### 1. 듣고 배우기
도메인 전문가, 현업 전문가에게 상세한 내용을 듣고 정리합니다.

### 2. 개념 식별 (CONCEPT IDENTIFICATION)
도메인에서 핵심적인 개념을 나타내는 단어를 정리합니다.

### 3. 관계 정의 (연결 고리 찾기)
각 개념의 관계를 정의합니다.
어떤 개념에서 어떤 개념이 필요한지 ...

### 4. 속성 및 기본 행위 명시 ( ATTRIBUTE AND BEHAVIOUR )
각 개념의 기본 동작과 속성을 명시하고 정리합니다.

### 5. 시각화 (VISUALIZING A DOMAIN MODEL)
그림을 그려 모두가 쉽게 이해할 수 있도록 정리합니다.
복잡한 UML 다이어그램이 필요한게 아닌, 단순 개념과 동작을 BOX 형태로 구성하여
모두가 알아볼 수 있도록 시각화합니다.

## 도메인 전문가가 없을 경우 ?
지금 당장 완벽한 대답을 해줄 수 있는 도메인 전문가가 없는 경우 ( 스타트업, 새로 출시되는 도메인 )
- 창업자, 기획자, PO를 가상의 도메인 전문가로 활용합니다.
- 모든 참여자가 도메인 전문가가 되어 의견을 나눕니다.
- 다양한 탐색과 집단적인 학습이 필요합니다. - 이벤트 스토밍 (함께 모여 워크샵 형태로 포스트잇을 붙이며 이야기를 나누는 것)
- 가설 기반의 점진적인 모델링 진행
- 보편 언어의 의식적인 구축과 진화 ( 하나의 개념을 다른 용어로 표현하는 경우는 없어야 합니다. )
- 베타 테스터, 초기 사용자와 적극적인 소통 필요
- **'도메인 모델'의 발전과 진화 공유를 적극적으로 해야 합니다.**

+ **경쟁 업체 서비스 분석** (bench marking)
